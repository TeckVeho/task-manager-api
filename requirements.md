# タスク管理API 要件

## 背景

チーム内でタスクを管理するための REST API を構築する。
認証付きで、タスクにカテゴリを付けたりメンバーにアサインできる機能が必要。

## 必要な機能

### 1. ユーザー管理
- ユーザーの新規登録（email, name, password）
- ログイン（JWT トークンを返す）
- ユーザー一覧の取得
- ユーザー情報の更新・削除

### 2. タスク管理
- タスクの作成（title, description, priority, status）
- タスク一覧の取得（status でフィルタリングできる）
- タスクの詳細取得
- タスクの更新・削除

### 3. カテゴリ管理
- カテゴリの作成（name, color）
- カテゴリ一覧の取得
- カテゴリの更新・削除

### 4. タスクアサイン機能
- タスクにユーザーをアサインする
- タスクにカテゴリを紐付ける
- 自分にアサインされたタスク一覧を取得する

## 技術要件

- データはインメモリ（`Map<string, T>`）で管理（DB 不要）
- 認証が必要なエンドポイントは JWT ベアラートークンで保護
- バリデーションエラーは適切なメッセージを返す
- 各機能に supertest を使ったテストを書く

## 注意

機能4（タスクアサイン）は機能1（ユーザー管理）と機能2（タスク管理）の
実装完了後でないと実装できない。
機能1〜3 は互いに独立しているため、並列で実装してよい。
