# Orchestrate Worktrees

要件またはタスクリストを受け取り、依存関係を分析して **implementor subagent** に並列委任し、マージまで管理する。

## フェーズ0：要件確認

タスク分解の前に、要件の曖昧な点・前提条件をユーザーに確認する。

**確認すべき観点：**
- **スコープ** — どこまでが今回の実装範囲か（例：認証は必要か、削除は論理削除か）
- **優先順位** — 機能間の優先度や段階リリースの有無
- **技術制約** — 使用禁止のライブラリ・既存コードとの整合性
- **完了定義** — 何をもって「完了」とするか（テスト基準など）
- **依存・連携** — 外部サービスや既存モジュールとの連携要否

**質問のルール：**
- 要件から明らかに読み取れることは質問しない
- 全ての質問を一度にまとめて提示する（往復を減らす）

**出力フォーマット：**

```
## 要件確認

以下の点を確認させてください：

1. [質問]
2. [質問]
...
```

全ての確認が取れてからフェーズ1へ進む。

## フェーズ1：タスク分解

確認済みの要件をもとにタスクリストに分解する。タスクリストがすでに渡されており要件確認も不要な場合はスキップ。

**分解の基準：**
- 1タスク = 独立してテスト・レビュー可能な単位（1 PR 相当）
- 1 subagent が1セッションで完結できる粒度
- 共有インフラ（DB定義・API型・ルーティング設定）は**最初のタスク**にまとめる

**出力フォーマット：**

```
## タスク分解案

1. [タスク名] - [1行の概要]
2. [タスク名] - [1行の概要]
...

このタスク分解で進めてよいですか？変更があれば指示してください。
```

ユーザーの承認後にフェーズ2へ進む。

## フェーズ2：タスク分析

各タスクについてコードベースを `grep` とセマンティック検索で調査し、変更対象ファイルを列挙する。

**出力フォーマット：**

```
## タスク分析結果

### タスク: [タスク名]
- 変更対象ファイル:
  - src/path/to/file.ts
- 他タスクとの重複: なし / あり（タスクXの○○と競合）
```

## フェーズ3：依存関係マッピングと実行計画

分析結果をもとに依存グラフを構築する。

**依存の判断基準：**
- 同じファイルを変更する → 順列必須
- 一方が他方の出力（関数・型・API）を呼び出す → 提供側を先に実行
- 完全に別ファイル・別モジュール → 並列実行可
- ホットスポット（`package.json`, routing, DB migration）→ 常に順列

**出力フォーマット：**

```
## 実行計画

### 🔀 並列グループ A（同時実行）
- [タスク名1]  →  変更: src/...
- [タスク名2]  →  変更: src/...

### ⏩ 順列: グループA完了後
- [タスク名3]  →  変更: src/...
  依存理由: タスク名1 が変更する XX を参照するため

---
承認しますか？ [y/n]
```

ユーザーの承認後にフェーズ4へ進む。

## フェーズ4：Subagent への並列委任

承認後、**並列グループ内のタスクを implementor subagent に同時委任する**。

各 subagent へのプロンプト形式：

```
以下のタスクを実装してください。

## タスク
[タスク名]: [詳細説明]

## 変更対象ファイル（参考）
- src/path/to/file.ts

## 完了条件
1. 実装が完了すること
2. 既存のテストが全て通過すること
3. 新機能のテストを追加すること

実装が完了したら結果を報告してください。
```

**注意：**
- 並列グループ内は `is_background: true` で同時起動
- 1グループの並列数は最大4つ
- 次グループは前グループの全 subagent 完了を確認してから委任する

## フェーズ5：マージ管理

各 subagent 完了後、以下の順序でマージする。

```bash
# 1. 最初のブランチをマージ
git checkout main
git merge --no-ff agent/[slug-1]

# 2. 残りのブランチを rebase
git -C .trees/[slug-2] rebase main || echo "コンフリクト: .trees/[slug-2] で解消してください"

# 3. テスト確認後、次をマージ
git checkout main
git merge --no-ff agent/[slug-2]
```

rebase でコンフリクトが出た場合は、該当 worktree で implementor subagent に解消を依頼する：
`「main への rebase でコンフリクトが発生しました。解消してください」`

全グループのマージ完了後、順列グループのタスクをフェーズ3と同じ手順で委任する。
